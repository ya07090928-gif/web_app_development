# PRD - 任務管理系統

## 1. 專案概述

### 背景與動機
大學生通常需要同時兼顧課業、報告、社團活動甚至是打工作業，生活節奏緊湊，導致經常面臨時間管理困難或忘記重要待辦事項的問題。為了解決這個痛點，本提案計畫開發一款簡單直覺的任務管理系統，協助他們集中記錄與追蹤所有的任務。

### 目標用戶
大專院校在學學生（大學生）。

### 核心價值主張
透過直覺易用的介面，幫助大學生快速建立、組織及追蹤日常任務，同時允許學生根據自己的主觀認知自訂任務的優先順序與重要性，進而有效管理時間、減輕記憶負擔，提升學習與生活效率。

---

## 2. 功能需求

以下為本系統的主要功能清單與使用者故事（User Story）：

1. **新增任務**
   - 作為一名大學生，我希望能夠快速新增任務，以便隨時記錄課堂上交代或突然想到的待辦事項。
2. **編輯任務**
   - 作為一名大學生，我希望能夠修改已經建立好的任務內容，以便在任務需求或細節變動時，保持資訊的正確性。
3. **刪除任務**
   - 作為一名大學生，我希望能夠刪除已經作廢或不小心建錯的任務，以便保持任務清單的乾淨整潔。
4. **標記完成狀態**
   - 作為一名大學生，我希望能夠將已做完的任務標記為「完成」，以便清楚看見自己目前的進度，並獲得完成任務的成就感。
5. **自行標記重要程度**
   - 作為一名大學生，我希望能夠為每一個任務設定不同的重要程度標籤（例如：高、中、低），以便在繁雜的任務中知道應該先處理哪一項。

---

## 3. 非功能需求

### 技術限制
- 後端框架：**Flask** (Python)
- 模板引擎：**Jinja2**
- 資料庫：**SQLite**

### 效能與安全考量
- **效能**：系統頁面載入與操作（如：標記完成、新增）反應時間應在 1 秒以內，確保流暢的使用者體驗。
- **安全與可靠性**：需防範基本的 SQL Injection 等常見網頁弱點；確保本機 SQLite 資料正確性與完整性，避免操作中斷導致資料庫損壞。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 說明 | 包含功能 |
| :--- | :--- | :--- |
| **Must Have** | 產品能運作的最小核心必備功能 | 新增任務、刪除任務、標記完成狀態、瀏覽任務清單 |
| **Should Have** | 重要但若無也不會導致產品無法使用的核心擴充 | 自行標記重要程度、編輯任務 |
| **Nice to Have** | 未來的加分項目，可以在後續迭代加入 | 任務到期日設定、任務分類或標籤、推播提醒通知 |
